### 1. [RegScan] OMOP RxNorm 기반 Decomposer 사전 검증 스크립트 작성 + 실행 (P2 — 완료)

#### 1-1. 배경

4/14에 구현한 `decomposer.py`의 3개 사전(`SALT_FORMS`, `REF_NORMALIZATION_MAP`, `FORMULATION_TOKENS`)은 MFDS 마스터 CSV 기반 임의 정의.
OMOP CDM의 RxNorm vocabulary(Athena 다운로드)로 교차 검증하여 누락/오류를 보정하기로 한 계획의 실행.

#### 1-2. 스크립트

- **파일**: `scripts/validate_decomposer_vocab.py`
- **데이터 소스**: `data/omop/CONCEPT.csv` (838MB, 6.4M rows)
- **처리 방식**: 스트리밍 1-pass로 RxNorm Ingredient(36,356건) / Precise Ingredient(3,658건) / Dose Form(258건) 추출 후, PI 이름에서 Ingredient 이름을 빼서 salt 토큰을 역추출

```
Usage: python scripts/validate_decomposer_vocab.py
```

**우리에만 있는 21개 (RxNorm 미등재)**:
`aluminium`, `ammonium`, `bicarbonate`, `diethanolamine`, `ethanolamine`, `gentisate`, `hemioxalate`, `hemisuccinate`, `hemitartrate`, `heptanoate`, `hexanoate`, `isobutyrate`, `isovalerate`, `lithium`, `napadisylate`, `phthalate`, `subacetate`, `subsalicylate`, `triacetate`, `triflutate`, `trihydrochloride`
→ RxNorm은 미국 중심(USP), 우리 마스터는 글로벌 INN 기반이므로 차이는 자연스러움. 삭제 불필요.

##### REF_NORMALIZATION_MAP (현재 21개 치환 규칙)

| Key (변이형) | Value (표준형) | Key in RxNorm | Value in RxNorm |
|-------------|---------------|:---:|:---:|
| besylate | besilate | OK | OK |
| hcl | hydrochloride | OK | OK |
| disodium | sodium | OK | OK |
| dimeglumine | meglumine | OK | OK |
| trisodium | sodium | OK | OK |
| trometamol | tromethamine | OK | OK |
| olamine | ethanolamine | OK | -- |
| 나머지 14개 | (표준형) | -- | OK |

- Key가 RxNorm에 있는 7개: RxNorm에서도 변이형을 실제 사용 → 치환 규칙 유효성 확인
- Value(표준형)는 거의 전부 RxNorm에 존재 — `ethanolamine`만 미등재 (RxNorm이 `olamine`을 그대로 사용)
- **조치**: `olamine → ethanolamine` 치환 방향 재검토 필요. RxNorm 기준이면 `ethanolamine → olamine`이 맞을 수 있으나, USP/INN 기준에서는 `ethanolamine`이 표준. 현행 유지.

##### FORMULATION_TOKENS (현재 29개)

| 구분 | 건수 |
|------|------|
| RxNorm Dose Form에서 확인 | 9개 (31%) |
| 마스터 고유 (RxNorm 미등재) | 20개 (69%) |

**RxNorm 확인된 9개**: `chewable`, `coated`, `delayed release`, `enteric coated`, `extended release`, `granule`, `granules`, `powder`, `sustained release`

**RxNorm에 없는 20개** (마스터 고유):
`micronized`, `liposomal`, `pegylated`, `nanocrystal`, `lyophilized`, `lyophilisate`, `concentrate`, `film coated`, `dispersible`, `controlled release`, `modified release`, `coated granules`, `coated powder`, `concentrate powder`, `concentrate granule(s)`, `enteric pellets`, `extended release microspheres`, `dried extract`, `soft extract`

→ RxNorm Dose Form은 "Oral Tablet", "Injectable Solution" 등 최종 제형 중심. 우리 토큰은 **원료 물리 형태**(micronized, granules)나 **가공 방식**(lyophilized, pegylated)이므로 분류 기준 자체가 다름. 누락이 아닌 체계 차이.

**RxNorm Dose Form에서 발견된 추가 후보 키워드**:
`effervescent`(2), `sublingual`(3), `implant`(1) — 마스터에서 실제 등장하는지 확인 후 추가 검토

4. **FORMULATION_TOKENS는 체계 차이** — RxNorm Dose Form과 분류 기준이 근본적으로 다름. 우리 토큰은 원료 물리 형태 중심이므로 OMOP 대비 부족이 아닌 상호 보완 관계
5. **Precise Ingredient 미분해 845건/3,658건** (23%) — 대부분 바이오의약품(-mab, -nib 접미사가 그대로 PI명), 복합 salt, 방사성 동위원소(F-18 등)

### 2. [RegScan] SALT_FORMS RxNorm 기반 확장 실험 + 구제율 검증 (P2 — 완료)

#### 2-1. 실험 설계

RxNorm에서 빈도 ≥ 3으로 검출된 45개 누락 salt 후보를 SALT_FORMS에 반영하고 구제율 변화를 측정.

#### 2-2. 1차 시도: 전수 투입 (실패)

45개 중 비-salt 제외 후 20개 + REF_NORMALIZATION_MAP 2개를 추가.

**결과: regression 발생** — unmatched +5건, decomposed_base_fallback -7건

**원인 분석** (7건 전수 역추적):
- `furoate` → "Fluticasone Furoate Micronized", "Mometasone Furoate Micronized" — furoate가 salt로 추출되어 base_key가 "fluticasone"으로 변경 → 마스터에 "fluticasone furoate"로만 등재되어 있어 매칭 실패
- `bromide` → "Aclidinium Bromide Micronized", "Tiotropium Bromide Hydrate" — 동일 패턴

**핵심 발견**: 할로겐화합물(chloride, bromide, iodide, fluoride) 및 특정 에스테르(furoate, dipropionate, pivalate, myristate, mononitrate)는 **약물명의 일부**이지 분리 가능한 염이 아님. SALT_FORMS에 넣으면 과잉 분해로 오히려 매칭률이 하락.

#### 2-3. 2차 시도: 안전 토큰만 선별 (성공)

과잉 분해 위험 9종 제거, 순수 염/에스테르/수화물 12종만 추가:

| 분류 | 추가 토큰 |
|------|----------|
| 유기산염 | `pidolate`, `adipate`, `cocoate`, `picrate` |
| 에스테르 | `acetonide`, `cypionate` |
| 유기황산염 | `methylsulfate` |
| 수지 복합체 | `polistirex` |
| 무기염 | `pyrophosphate` |
| 수화물 변이 | `hexahydrate`, `tetrahydrate`, `decahydrate` |

REF_NORMALIZATION_MAP 추가: `tetrasodium→sodium`, `monohydrochloride→hydrochloride`
