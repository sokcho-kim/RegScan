```

2. python -m regscan.article.cite articles.md        # 출처 추가
3. python -m regscan.article.renderer articles.md    # HTML 렌더링
```

**`regscan/article/cite.py`** (신규):

## KHIRA-NER 프로젝트 점검 및 방향 결정

### 현황 분석

Phase 1 데이터셋 파이프라인은 끝까지 돌았으나, 양이 부족:
- 3,000문장 샘플링 → Gazetteer 라벨링 → GPT-5 검증(일부) → Entity Linking → 포맷 변환
- **최종: train 1,824 + test 365 = 2,189문장** (목표 3,000 대비 73%)
- 811문장(27%)이 엔티티 0건으로 제외됨

### 손실 원인
