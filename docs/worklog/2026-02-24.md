### 3. 변수명 누출 문제 발견

V4 기사를 브라우저에서 확인한 결과, LLM 인사이트 텍스트에 **팩트 변수명이 그대로 출력**되는 문제 발견.

| 기사에 나온 텍스트 | 원래 의도 |
|---|---|
| `fda_status_text 및 ema_status_text처럼` | "FDA 승인(2024-11-20) 및 EMA 승인(2025-06-27)처럼" |
| `국내는 mfds_status_text 상태이며` | "국내는 미허가 상태이며" |
| `copay_scenario_text대로 사실상 비급여` | "전액 비급여 구간으로…" |
| `context.d_day_text가 시사하듯` | "글로벌 승인 후 461일이라는 점이 시사하듯" |

**원인 분석**: 프롬프트 `BRIEFING_REPORT_PROMPT_V4`의 규칙 텍스트에서 변수명을 과도하게 반복 노출 (475행: "팩트 필드(fda_status_text, mfds_status_text 등)를 인용하라", 538행, 433행 등). LLM이 "인용하라"를 변수명 자체를 텍스트에 쓰라고 해석.

#### 4-1. 테스트 설계

두 가지 수정 전략을 비교:

| | Prompt A (Positive Framing) | Prompt B (Negative Rule) |
|---|---|---|
| **전략** | 데이터를 '원료'로 정의, '문장으로 치환하라' + Bad/Good 예시 | '변수명을 출력하지 마라' 금지 조항 추가 |

**Prompt A 핵심 패치:**
```
## Data Interpretation Guidelines
입력된 JSON 데이터는 기사 작성을 위한 '원료'입니다.
아래의 [치환 규칙]과 [예시]를 따라 데이터를 자연스러운 '문장'으로 변환하여 작성하십시오.

### 치환 규칙
1. 날짜 경과 데이터 → 시간의 흐름을 체감할 수 있는 '서술어'로 변환
2. 환자 비용 시나리오 → 구체적인 '재정적 상황'으로 묘사
3. 규제 기관 승인 상태 → 규제 기관의 '권위 있는 결정'으로 표현

### 작성 예시
[나쁜 출력] "fda_status_text에 따르면 승인되었고, d_day_text입니다."
[좋은 출력] "FDA가 2024년 11월 승인을 완료한 지 벌써 461일이 흘렀다."
```

**Prompt B 핵심 패치:**
```
6. 데이터 필드명 출력 금지: fda_status_text, d_day_text, copay_scenario_text 등
```
