### INN 정규화 엔진 강화 — 포괄적 염 사전 + Strict FDA Regex

크롤러 근본 수정 #1(USAN 접미사) 검증 과정에서 정규화 로직 전반을 재설계.

#### 변경 동기

DB 점검 결과 5건 대상 약물이 대소문자·USAN 접미사·염 형태 차이로 **2~4건씩 중복 존재** (총 1,386건 중 상당수 중복 추정). 기존 `SUFFIX_PATTERNS` 5개 규칙으로는 커버 불가.

#### 수정 파일 (4개)

| 파일 | 변경 내용 |
|------|-----------|
| `regscan/map/matcher.py` | `SUFFIX_PATTERNS`(5규칙) → `SALT_FORMS` frozenset(**122종** FDA/USP 기준) + `_salt_re` 컴파일 정규식. `normalize()` 재작성: 입체이성질체 접두사(`(s)-`, `s-`) → 괄호 → 콤마 → 염/에스테르(반복 제거) → FDA 4자리 접미사 → 정리 |
| `regscan/stream/therapeutic.py` | FDA 교차검색 regex `-[a-z]{3,5}$` → **`-[a-z]{4}$`** (정확히 4자리) |
| `regscan/db/loader.py` | `_normalize_inn()` classmethod 추가 — `IngredientMatcher.normalize()` 위임. `_upsert_drug`, `_get_drug_id`, `_upsert_drug_with_changes`, `normalize_existing_inns` 4곳의 `inn.lower().strip()` → `_normalize_inn()` 통일 |
| `regscan/scripts/verify_5drugs.py` | 5건 약물 검증 스크립트 신규. `--check` DB 현황, `--reset` DB 백업→초기화→전체 파이프라인, `--publish` 5건 기사, `--compare` 스냅샷 비교 |

#### 설계 원칙

1. **FDA 접미사 Strict**: `-[a-z]{4}$` — FDA 바이오의약품 식별자는 **정확히 4자리**. 3자리(`-mab`)·5자리(`-tinib`)의 USAN 어간 훼손 원천 차단
2. **염 사전 포괄적**: 하드코딩 땜빵 금지. FDA/USP 기준 산성염·유기산염·술폰산염·염기염·수화물 **122종** 일괄 수록
3. **입체이성질체 전처리**: `(s)-`, `(r)-`, `s-`, `d-`, `l-` 등 괄호/bare 형태 모두 정규식으로 선제 제거 후 염 매칭
4. **정규화 통합**: DB 저장(`loader.py`)과 매칭(`matcher.py`)이 동일한 `normalize()` 사용 → 중복 원천 차단
