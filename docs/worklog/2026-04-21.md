#### 9. OMOP/RxNorm 데이터 삭제 (2.8GB)

**활용 이력:**
- Decomposer v1.0.0 개발 시 SALT_FORMS, FORMULATION_TOKENS 사전을 OMOP RxNorm과 교차 검증 (`scripts/validate_decomposer_vocab.py`)
- CONCEPT.csv (838MB) + CONCEPT_RELATIONSHIP.csv (1.9GB) + DRUG_STRENGTH.csv (151MB)
- Athena(athena.ohdsi.org)에서 다운로드한 RxNorm vocabulary

**삭제 이유:**
- 런타임에서 `data/omop/`를 참조하는 코드 **없음** (grep 확인)
- 검증 스크립트(`validate_decomposer_vocab.py`)에서만 1회성 사용
- Decomposer 사전 검증은 이미 완료되어 Logic Frozen 상태
- **디스크 100%** 문제의 주범 (233GB 중 2.8GB)
- 필요시 Athena에서 재다운로드 가능

**교훈:** 대용량 참조 데이터는 검증 완료 후 로컬에서 제거하고, 필요시 재다운로드하는 정책 필요. `.gitignore`에 이미 포함되어 있어 원격에는 영향 없음.
