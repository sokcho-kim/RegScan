#### 신규 파일

#### 수정 파일

| 파일 | 변경 내용 |
|------|----------|
| `regscan/db/loader.py` | `upsert_hira_reimbursement()`, `upsert_mfds_permit()` 메서드 추가 |

#### HIRA 워커 (`hira_worker.py`)

- **데이터 소스**: 공공데이터포털 `B551182/pharmacyInfoService` — 의약품 급여·비급여 목록 및 상한금액
- **인증**: `DATA_GO_KR_API_KEY` (MFDS와 동일 키)
- **구성 요소**:
  - `HIRADrugPriceClient` — async API 클라이언트 (재시도 3회, rate limit 0.2초, 429 backoff)
  - `parse_price_item()` — API 응답 → 정규화 (status/price_ceiling/ingredient_code 추출)
  - `HIRAReimbursementWorker` — 3가지 모드
    - `full`: 전체 급여목록 페이지네이션 수집 (초기 구축용)
    - `by_inn`: DB의 INN 목록 기반 개별 조회 (일일 갱신용)
    - `--dry-run`: DB 저장 없이 API 응답 확인
- **급여 상태 판정**: PAY_TP_NM 필드 → reimbursed / not_covered / deleted / not_found

#### MFDS 워커 (`mfds_worker.py`)

- **데이터 소스**: 기존 `MFDSClient` + `MFDSPermitParser` 재활용
- **구성 요소**:
  - `MFDSPermitWorker` — 3가지 모드
    - `delta`: 최근 N일 변경분만 수집 (허가일 기준 cutoff, 기본 7일)
    - `full`: 전체 수집 (max_items 제한 가능)
    - `by_inn`: INN 목록 기반 조회
- **delta 전략**: API에 날짜 필터 없으므로 최신순 페이지네이션 → 허가일 < cutoff 시 중단

#### DBLoader 확장

- `upsert_hira_reimbursement()` — ingredient_code/query_inn 기준 hira_reimbursements upsert
- `upsert_mfds_permit()` — INN 기준 regulatory_events(agency='mfds') upsert, 날짜 파싱 포함

#### CLI 사용법

```bash

# MFDS 허가정보 — API 응답 확인
```

### 4. [RegScan] HIRA 워커 dry-run 결과 + 기존 접근법 재확인

#### HIRA API dry-run 결과

| 시도 | 결과 | 원인 |
|------|------|------|
| `type=json` 파라미터 포함 | 403 Forbidden | **XML 전용 API**, JSON 미지원 |
| `type=json` 제거 (XML) | 200 OK | 정상 응답 |
| 영문 INN 검색 (keytruda) | totalCount=0 | **한글 품목명만 검색 가능** |
| 한글 품목명 (타이레놀) | totalCount=25 | 정상 |
| 파라미터 없이 전체 조회 | totalCount=0 | **검색 파라미터 필수** (전체 목록 제공 안 함) |

→ 이 API는 **품목명(한글) 기반 개별 조회** 전용이며, 전체 덤프용이 아님.

#### 기존 HIRA 데이터 구축 방식 (이미 구현되어 있었음!)

기존 1,099건의 `hira_reimbursements`는 **공공 API가 아니라 로컬 파일 기반 IngredientBridge**로 적재된 것:

```
기존 흐름:
  로컬 파일 (수동 다운로드)
    ├─ data/bridge/yakga_ingredient_master.csv     (59,631건 — 약품주성분마스터)
    ├─ data/hira/drug_prices_20260204.json         (59,004건 — HIRA 약가)
    └─ data/bridge/건강보험심사평가원_ATC코드_매핑_목록.csv  (21,953건)
         ↓
  IngredientBridge.lookup(inn)  — 3단계 매칭 (정규화→ATC→직접)
         ↓
  DomesticImpactAnalyzer._enrich_hira()
         ↓
  DBLoader._upsert_hira()  → hira_reimbursements 테이블
```

**매칭 성과**: MFDS 44,035건 중 33,194건(75.4%) HIRA 연결 성공

#### 공공 API vs 로컬 파일 비교

| 항목 | 공공 API (dgamtCrtrInfoService) | 로컬 파일 (IngredientBridge) |
|------|-------------------------------|---------------------------|
| 전체 덤프 | X (검색 파라미터 필수) | O (59,004건 일괄) |
| 실시간성 | O (실시간 반영) | X (수동 다운로드 필요) |
| 검색 방식 | 한글 품목명만 | INN(영문) 정규화 매칭 |
| 상한가 | 응답에 mxCprc 없음 (확인 필요) | O (price_ceiling 포함) |
| 자동화 | API 호출 자동화 가능 | 파일 갱신 주기 수동 |

#### 결론: 하이브리드 전략

1. **일괄 데이터**: 기존 IngredientBridge 유지 (로컬 파일이 59K건 전체 커버)
2. **증분 갱신**: 공공 API로 신규/변경 약물만 조회 (한글 품목명 기반)
3. **워커 역할 변경**: 전체 수집 → 기존 IngredientBridge 파일 갱신 자동화 + API로 신규 약물 보완

→ HIRA 워커를 **로컬 파일 갱신 + API 보완** 하이브리드로 리팩터링 필요

### 5. [RegScan] DrugPriceCollector 구축 — 적용약가파일 자동 수집

HIRA 적용약가파일(Excel)을 자동 다운로드하고 JSON으로 변환하는 워커 구축.

#### 신규 파일: `regscan/workers/drug_price_collector.py`

**3가지 기능:**

1. **Playwright 다운로더** — biz.hira.or.kr에서 최신 약가 Excel 자동 다운로드
   - 여러 셀렉터 전략으로 링크 탐색 (적용약가 텍스트, xlsx 링크, iframe 내부)
   - 파일명에서 날짜 자동 추출
2. **Excel → JSON 변환기** — pandas + openpyxl로 컬럼 매핑 변환
   - 31개 컬럼 매핑 (상한가→price_ceiling, 주성분코드→ingredient_code 등)
   - 날짜 컬럼 YYYY-MM-DD 정규화
   - NaN → None 변환
3. **변경 감지** — SHA256 해시 비교로 기존 파일과 동일하면 스킵

**검증 결과:**
- `--convert-only` 테스트: 기존 Excel → JSON 59,004건 변환 성공 (52.1MB)
- 컬럼 매핑 정확 (ingredient_code: 636401ACH, price_ceiling: 470.0, 급여기준: 급여)

**CLI 사용법:**
```bash
# 전체 파이프라인 (다운로드 + 변환 + 변경감지)
python -m regscan.workers.drug_price_collector

# 기존 Excel → JSON 변환만
python -m regscan.workers.drug_price_collector --convert-only data/hira/약가파일.xlsx

# 다운로드만
python -m regscan.workers.drug_price_collector --download-only

# GUI 모드 (디버깅)
python -m regscan.workers.drug_price_collector --no-headless
```

**남은 작업:**
- biz.hira.or.kr 실제 접속 테스트 (사이트 구조에 따라 셀렉터 조정 필요)
- 스케줄러 등록 (daily check → 신규 파일 감지 시 실행)
