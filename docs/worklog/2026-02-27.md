# 작업일지 - 2026-02-27

## 작업 내역

### C-3: 뉴스 → LLM 연동 파이프라인 구현

V4 기사 파이프라인에 최근 뉴스 맥락을 주입하여 기사 품질 향상. C-2 가격 스펙트럼 패턴과 동일한 구조로 구현.

#### 아키텍처

```
RSS fetch (Endpoints + FiercePharma + FierceBiotech)
  → regscan/news/fetcher.py (NewsArticle dataclass)
  → regscan/news/matcher.py (IngredientMatcher.SYNONYMS 기반 INN 매칭, 161개 토큰)
  → publish_articles.py: 파이프라인 시작 시 1회 수집 → impact._news_cache 주입
  → llm_generator.py: _get_recent_news() → data["recent_news"]
  → prompts.py: 뉴스 인용 규칙 (팩트 우선, 1~2건 자연스러운 인용)
```

#### 신규 파일 (3개)

| 파일 | 설명 |
|------|------|
| `regscan/news/__init__.py` | 빈 패키지 |
| `regscan/news/fetcher.py` | RSS 수집기 — `fetch_news()`, xml.etree 기반 파싱, 날짜 필터링 |
| `regscan/news/matcher.py` | INN↔뉴스 매칭 — `match_news_to_inns()`, 단어 경계 + 한글 포함 검색 |

#### 수정 파일 (4개)

| 파일 | 변경 내용 |
|------|-----------|
| `regscan/config/settings.py` | `ENABLE_NEWS_FETCH`, `NEWS_FETCH_DAYS_BACK`(7), `NEWS_FETCH_LIMIT`(5), `NEWS_FETCH_TIMEOUT`(15.0) |
| `regscan/report/llm_generator.py` | `_get_recent_news()` 메서드 추가 + `_prepare_drug_data_v4()`에 `recent_news` 필드 주입 |
| `regscan/report/prompts.py` | `★ 최근 뉴스 인용 규칙 ★` 블록 추가 (팩트 우선 원칙, 모순 시 무시) |
| `regscan/scripts/publish_articles.py` | `run_publish()` 시작 시 뉴스 1회 수집 → `news_cache` 구축 → `impact._news_cache` 주입 |

#### 설계 원칙

- **Graceful degradation**: RSS 실패 시 `logger.warning` 후 뉴스 없이 진행
- **팩트 우선**: 뉴스가 사전 계산 팩트와 모순되면 뉴스 무시
- **외부 의존성 없음**: feedparser 불필요 (xml.etree 사용)
