## 작업 내역

### [RegScan] 데이터 수집 갭 분석 — 타 레포 이식 대비

RegScan 데이터 수집 체계를 다른 레포에 이식하기 위해, 현재 16개 수집 소스를 정리하고 누락된 데이터 소스를 식별.

#### 현재 수집 중 (16개 소스)
- 글로벌 규제: FDA, EMA
- 국내 규제: MFDS, HIRA, MOHW, KDCA
- 임상시험: ClinicalTrials.gov, CRIS
- 산업/시장: KHIDI, ASTI, Health.kr
- 뉴스: Endpoints, FiercePharma, FierceBiotech
- 학술: bioRxiv/medRxiv
- 참조 DB: OMOP/RxNorm, ATC, HeMonc, HIRA 약가마스터

#### 추가 수집 검토 대상 (우선순위순)

| 순위 | 소스 | 핵심 이유 |
|-----|------|----------|
| 1 | FDA Orange Book (특허) | 제네릭 진입 시점 예측 |
| 2 | MFDS 안전성 서한/회수 | 국내 규제 사각지대 |
| 3 | NICE/CADTH HTA 결정 | 급여 가능성 예측 선행지표 |
| 4 | PMDA (일본) | IRP 참조국 |
| 5 | 건보심/국회 법안 | 국내 정책 파이프라인 완성 |
| 6 | DART/EDGAR 공시 | 라이선스 딜 선행지표 |
| 7 | 학회/진료지침 | 적응증 확대 경로 |
| 8 | 글로벌 약가 | IRP 비교 벤치마크 |

- **결과물**: [`docs/research/data-collection-gap-analysis.md`](../research/data-collection-gap-analysis.md)

### [RegScan] FDA Orange Book 수집기 구현 — 특허/독점권 만료 데이터

갭 분석 우선순위 1번 소스 구현 완료.

#### 구현 내역

- `FDAOrangeBookIngestor` — 전체 Orange Book 다운로드 (product + patent + exclusivity)
- `FDAPatentExpiryIngestor` — patent + exclusivity를 product와 조인하여 INN별 특허 만료일 반환
- `OrangeBookClient` — ZIP 다운로드 + 틸드(~) 구분 파일 파싱

#### 수집 데이터

| 항목 | 건수 |
|------|------|
| 제품 (NDA+ANDA) | 48,083 |
| 특허 | 20,858 |
| 독점권 | 2,054 |
| 2026 만료 특허 | 1,547 |
| 2027 만료 특허 | 2,102 |

#### 검증 결과

| 약물 | 브랜드 | 최후 특허 만료 |
|------|--------|-------------|
| SEMAGLUTIDE | OZEMPIC | Sep 29, 2027 |
| LENALIDOMIDE | REVLIMID | Mar 8, 2028 |
| APIXABAN | ELIQUIS | Nov 22, 2040 |
| TOFACITINIB | XELJANZ XR | Sep 14, 2034 |

#### 제한사항

- Orange Book = NDA(소분자) + ANDA(제네릭)만 수록
- BLA(생물의약품, e.g. pembrolizumab/KEYTRUDA)는 **Purple Book** 별도 수집 필요 → 추후 구현

#### 변경 파일

| 파일 | 변경 |
|------|------|
| `regscan/ingest/orange_book.py` | **신규** — Orange Book 수집기 |
| `regscan/ingest/__init__.py` | **수정** — 모듈 등록 |
| `regscan/config/settings.py` | **수정** — `ENABLE_FDA_ORANGE_BOOK` 토글 추가 |
