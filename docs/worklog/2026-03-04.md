# 작업일지 - 2026-03-04

## 작업 내역

### semaglutide FDA 승인일 None 원인 조사 및 수정

전일 검증에서 발견된 semaglutide FDA approval_date=None 이슈의 근본 원인을 추적하여 수정.

#### 원인 분석

`_search_fda_pharm_class` (therapeutic.py)가 `fda_data` 생성 시 `FDADrugParser`를 사용하지 않고 수동 구성 → `submission_status_date` 미포함. semaglutide는 "GLP-1 Receptor Agonist"로 pharm_class 검색에서 이미 잡혀 `fda_data`가 존재 → `_enrich_with_fda_crossref`의 `not drug.get("fda_data")` 조건에 의해 스킵 → 날짜 없는 fda_data가 최종까지 전파.

#### 코드 수정 (2개 파일)

| 파일 | 변경 |
|------|------|
| `regscan/stream/therapeutic.py` | `_search_fda_pharm_class`에 `FDADrugParser._extract_latest_submission()` 도입 — ORIG+AP 우선순위로 `submission_status_date` 등 5개 필드 추출 |
| `regscan/map/global_status.py` | `_build_fda_approval`에 fallback 추가 — `submission_status_date` 누락 시 `submissions` raw에서 자동 추출 (belt & suspenders) |

### FDA approval_date NULL 약물 일괄 Backfill

`_search_fda_pharm_class` 버그는 공통 모듈이므로, semaglutide 외 대량의 약물이 동일 이슈로 FDA 승인일 누락 상태. DB 전수 조사 후 일괄 복구 실행.

#### Backfill 스크립트

- **경로**: `regscan/scripts/backfill_fda_dates.py`
- FDA API에서 INN으로 재조회 → `FDADrugParser._extract_latest_submission()` (ORIG+AP 우선순위) → DB 업데이트
- `--dry-run`, `--limit N` 옵션 지원

#### 결과

| 지표 | Before | After |
|------|--------|-------|
| FDA approval_date 보유 | 221건 (44.5%) | **497건 (100.0%)** |
| FDA approval_date NULL | 276건 (55.5%) | **0건 (0.0%)** |
| application_number 보유 | 0건 | **277건 (55.7%)** |

- 276건 전량 UPDATE, SKIP 0건, ERROR 0건
- semaglutide: `2017-12-05` (NDA209637) 정상 확인
- 부가 성과: `application_number`도 276건 동시 채워짐
