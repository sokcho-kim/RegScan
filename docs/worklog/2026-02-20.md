### 1. 인라인 아웃링크 시스템 (`_inject_outlinks`)

**구현**: 본문 텍스트에서 4가지 패턴 자동 감지 → 하이퍼링크 삽입.

| 패턴 | 링크 대상 | 예시 |
|------|----------|------|
| INN 약물명 첫 등장 | FDA 또는 EMA EPAR 페이지 | "Polatuzumab Vedotin" → FDA 링크 |
| "FDA...승인" 컨텍스트 | FDA Drug Approval | "FDA가 2023년 4월 정식 승인" → 링크 |
| "EMA...허가" 컨텍스트 | EMA EPAR | "EMA가 2020년 1월 조건부 승인" → 링크 |
| 임상시험 키워드 | CT.gov 스터디 페이지 | "3상 POLARIX 시험" → NCT04236141 |

- `seen` set을 섹션 간 공유하여 전체 기사에서 **각 링크 최대 1회**만 삽입
- `nct_id`, `source_urls`가 없으면 해당 패턴 스킵 (안전)

### 2. 타임라인 위젯 버그 수정

**문제**: 타임라인 위젯이 **항상 빈 상태**로 렌더링됨.

**근본 원인**: `_build_timeline_html()`이 `source_data["fda"]["approved"]` (중첩 구조)를 기대했으나, `DomesticImpact.to_dict()`는 `source_data["fda_approved"]` (플랫 구조)를 반환.

**수정**: 호환 레이어 추가
```python
fda = source_data.get("fda") or {
    "approved": source_data.get("fda_approved"),
    "date": source_data.get("fda_date"),
}
```

- `_build_flags_html()`에도 동일 패턴 적용

---

### 3. 태그 배지 버그 수정

**문제**: 희귀의약품/혁신치료제/EMA PRIME 태그 배지가 **항상 빈 상태**.

**근본 원인**: `_build_tag_badges()`가 `source_data["analysis"]["hot_issue_reasons"]`를 찾지만, `to_dict()` 출력에 `"analysis"` 서브키가 없음.

**수정**: `run_publish()`에서 `report_data["source_data"]`에 직접 주입
```python
report_data["source_data"]["analysis"] = {
    "hot_issue_reasons": getattr(impact, 'hot_issue_reasons', []) or [],
}
```

### 4. HTML-only 재렌더 모드 (`--render-only`)

**목적**: 매 수정마다 LLM 20건 재생성(~5분) → HTML만 재렌더(~2초).

**구현**: `_run_render_only()` 함수
- 기존 JSON 로드 → `generate_article_html()` → HTML 저장
- LLM·DB·API 호출 **전혀 없음**
- CLI: `python -m regscan.scripts.publish_articles --render-only`
- 151건 HTML 재생성: **~1초**

### 5. JSON 메타데이터 보강

**목적**: `--render-only` 모드에서 링크/점수/태그 등을 복원하려면 JSON에 메타데이터 필요.

**추가 필드**:
```
_source_urls: {fda: "...", ema: "..."}
_nct_id: "NCT04236141"
_score: 60
_hot_issue_reasons: ["FDA 승인", "EMA 승인", ...]
_therapeutic_areas: ["oncology"]
_known_inns: ["Zanidatamab-hrii", "Zolbetuximab", ...]
source_data: {fda_approved: true, fda_date: "...", ...}
```

- LLM 경로 / skip-llm 경로 **공통 섹션**에서 JSON 재저장

### 6. LLM 프롬프트 v3.0 + 동적 헤딩

**프롬프트 4대 원칙 추가** (`regscan/report/prompts.py`):
1. **Dynamic Headings**: 고정 "글로벌 승인 현황" → 약물 특성 반영 동적 제목
2. **Visual Chunking**: 마크다운 표/불릿으로 데이터 구조화
3. **Contextual Translation**: 규제 용어 한국어 맥락 번역
4. **Lifecycle Angle**: 약물 생애주기 단계별 차별화된 분석 시각

**Few-Shot 3건 전면 교체** (동적 헤딩 + 표/불릿 포함)

**HTML 지원**:
- `_md_to_html()` 함수: 마크다운 표/불릿/bold → HTML 변환
- `BriefingReport` 데이터클래스에 `global_heading`, `domestic_heading` 필드 추가
- HTML 템플릿의 `<h2>` 태그가 동적 헤딩 변수 사용

### 7. 경쟁약 내부 링크 (`_inject_competitor_links`)

**구현**: 본문에 등장하는 경쟁약 INN에 해당 브리핑 페이지 링크 삽입.

**3-tier 매칭**:

| 순위 | 소스 | 링크 대상 |
|------|------|----------|
| 1순위 | DB 경쟁약 목록 (`_competitors`) | 내부 브리핑 HTML |
| 2순위 | 전체 브리핑 JSON 인덱스 | 내부 브리핑 HTML |
| 3순위 | ALL CAPS 패턴 (6자 이상) | CT.gov 검색 링크 |

- `_build_all_inns_index()`: 전체 JSON에서 INN→파일명 인덱스 구축 (모듈 레벨 캐시)
- `_get_all_inns_index()`: 캐시된 인덱스 반환 (render-only 시 1회만 구축)

### 8. POLARIX 버그 수정 (임상시험명 오탐)

**문제**: POLARIX(3상 임상시험 이름)가 ALL CAPS 패턴(6자 이상)에 매칭 → `intr=POLARIX` CT.gov 검색 링크 생성 → **결과 0건** (POLARIX는 intervention이 아님)

**수정**: `_inject_competitor_links()`에 임상시험 컨텍스트 감지 추가
```python
trial_context = re.compile(r'시험|trial|study|임상|연구', re.IGNORECASE)
context = text[ctx_start:m.start()] + text[m.end():ctx_end]
if trial_context.search(context):
    continue  # 스킵
```

- 앞뒤 30자 내에 "시험/trial/study/임상/연구" 있으면 약물이 아닌 시험명으로 판단
- POLARIX는 이제 `_inject_outlinks()`의 CT.gov 패턴으로만 매칭 → NCT04236141 정상 링크

### 9. JSON 메타데이터 일괄 패치 (`scripts/patch_json_metadata.py`)

**문제**: 151건 브리핑 JSON 중 **149건**이 구 형식 — `_source_urls`, `_score`, `source_data` 전부 없음. `--render-only`로 HTML 재렌더해도 FDA/EMA 링크, 타임라인, 태그 배지가 모두 빈 상태.

**구현**: DB에서 메타데이터만 읽어 JSON에 패치하는 독립 스크립트
- LLM/CT.gov/EMA API 호출 없음
- DB → `source_urls`, `score`, `source_data`, `hot_issue_reasons`, `therapeutic_areas` 추출
- 결과: **113건 패치 성공**, 38건 스킵 (DB 미매칭)

## 이슈

- **ALL CAPS 패턴 3순위 매칭**: POLARIX 외에도 KEYNOTE, DESTINY 등 임상시험 이름이 오탐될 가능성. 현재 컨텍스트 감지(30자)로 대응하지만, 알려진 시험명 블랙리스트 추가 검토 가능.
- **`--render-only`에서 `global_heading`/`domestic_heading` 미복원**: 구 JSON에는 해당 필드 없어 기본값("글로벌 승인 현황" / "국내 도입 전망") 사용. LLM 재생성 시 동적 헤딩 적용됨.
- **JSON 패치 후에도 38건 source_urls 비어 있음**: DB에 해당 약물이 없거나 INN 매칭 실패. 파이프라인 재실행 시 해결 예상.
- **LLM 재생성 미완료**: 프롬프트 v3.0 적용되었지만 실제 기사 내용은 구 프롬프트 기준. `--skip-llm` 없이 재실행 필요.
