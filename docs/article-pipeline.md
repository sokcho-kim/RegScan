# 기사 생성 파이프라인 (Article Pipeline) 기술 문서

> 최종 수정: 2026-04-28

## 개요

RegScan의 기사 생성 파이프라인은 **5-Agent 2-Pass** 구조로, 수집된 규제/의약품 시그널을 전문지 수준의 기사로 변환한다.

**핵심 파일:**
- `regscan/article/pipeline.py` — 5-Agent 오케스트레이션 + 품질 검증
- `regscan/article/enrichment.py` — 시그널 엔리칭 (FDA, MFDS, HIRA, 크로스레퍼런스)
- `regscan/article/guardrails.py` — 전처리/후처리/사후검증 가드레일
- `regscan/article/run.py` — CLI 진입점

**실행:**
```bash
python -m regscan.article.run              # 기본 30일
python -m regscan.article.run --days-back 14
```

## 엔리칭 상세: 국내 크로스레퍼런스

**위치:** `enrichment.py` lines 855~998

글로벌 시그널(NICE_TA, KHIDI_GLOBAL_INFO 등)에서 약물 INN을 감지하면 자동으로 국내 데이터를 조회하여 시그널에 추가한다.

### 흐름

```
시그널 텍스트에서 INN 추출 (_extract_inns_from_text)
    │
    ├── _INN_TO_KR_NAME 매핑 (20+ 약물)
    │   예: PEMBROLIZUMAB → ["키트루다", "펨브롤리주맙"]
    │
    ▼
약물별 3종 조회:
    1. MFDS 허가 → fetch_mfds_permit_detail(inn)
    2. HIRA 급여 → fetch_hira_reimbursement(inn)
    3. 한국 임상 → _fetch_kr_clinical_trials(inn)
    │
    ▼
sig["fda_context"]에 태그 형태로 추가:
    [국내 크로스레퍼런스]
    [국내 허가: PEMBROLIZUMAB (키트루다/펨브롤리주맙)] 효능: ... | 용법: ...
    [국내 급여: PEMBROLIZUMAB] 급여 등재, 상한가 2,103,620원
    [국내 임상시험: PEMBROLIZUMAB] 3건 진행 중
```

### 기자 프롬프트 연동

`pipeline.py` line 348에서 기자에게 크로스레퍼런스 활용 의무를 지시:
- `[국내 허가:`, `[국내 급여:`, `[국내 임상시험:` 태그가 있으면 본문에 반드시 반영
- 무시 시 불합격 처리

## 품질 검증 상세

현재 품질 관리는 LLM 단계와 코드 단계로 나뉜다.

### LLM 단계

- 편집장: 등급, 방향, 핵심 근거 지정
- 기자: 등급별 기사 포맷 생성
- 팩트체커: 하드 룰 위반/수치/날짜 검토
- 구조검증기: 누락 슬롯 보강
- 편집자: 제목-리드-본문 메시지 정렬

### 코드 단계

- `post_process_article()`
  - 금지 문장 삭제
  - 기관명 `(이하 약어)` 정규화
  - 메타 코멘트/불완전 문장 제거
- `validate_article_grounding()`
  - 법안 기사에서 원문에 없는 날짜 탐지
  - 법안 기사에서 원문에 없는 조문 탐지
  - 제목의 `N건`과 본문 사례 수 불일치 탐지
  - 위반 시 등급 다운그레이드

### 현재 핵심 검증 항목

| 항목 | 대상 등급 | 조건 | 실패 메시지 |
|------|-----------|------|-------------|
| 길이 | A/B | `len(body) < min_len` | 등급 다운 가능 |
| structural_info 누락 | A | 구조검증기에서 NOT_FOUND 2개 이상 | A→B 다운 |
| 법안 날짜 불일치 | regulation | 기사 날짜가 원문 데이터에 없음 | 등급 다운 + 경고 |
| 법안 조문 불일치 | regulation | 기사 조문이 원문 데이터에 없음 | 등급 다운 + 경고 |
| 제목 범위 불일치 | 전체 | 제목 `N건` < 본문 사례 수 | 등급 다운 + 경고 |

## 알려진 한계 및 개선 방향

### 한계: 프롬프트 지시만으로는 헛발을 막기 어렵다

크로스레퍼런스 데이터가 시그널에 정상 주입되어도, 기자 에이전트가 확률적으로 무시하는 문제가 있다.

**실제 결과 (2026-04-28):**
- 기사 3편 중 1편(#3 survodutide)에서 문장 잘림 + 크로스레퍼런스 미반영
- 프롬프트에 "반드시 활용하라 + 무시하면 불합격" 추가했으나 완전한 해결 아님

추가로 아래 유형의 헛발이 실제로 확인됐다.

- 법안 기사에서 원문에 없는 날짜/조문을 조합해 서술
- 제목은 `2건`인데 본문은 패턴 기사처럼 더 많은 사례를 확장
- 사용자 보고용 요약이 실제 생성 파일과 어긋남

**원인**

- LLM이 시스템 프롬프트의 "의무" 지시를 확률적으로 무시
- 컨텍스트가 길수록 중간 메타데이터를 섞어 쓰는 경향
- 생성 이후 요약 레이어에 대한 별도 검증 부재

### 구조적 대안 후보

| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **템플릿 슬롯** | 기사 구조에 `[국내 비교]` 섹션을 하드코딩, 엔리칭 데이터 직접 삽입 | 100% 보장 | 기사 자연스러움 감소 |
| **2-pass 생성** | 1차 기사 작성 후, 크로스레퍼런스 데이터를 주며 리라이트 | 자연스러움 유지 | API 비용 2배 |
| **후처리 삽입** | 완성된 기사에 크로스레퍼런스 단락을 프로그래밍적으로 삽입 | 확실, 비용 없음 | 맥락 무시 가능 |
| **recency bias 활용** | 크로스레퍼런스를 프롬프트 최하단에 배치 | 간단 | 여전히 확률적 |
| **검증+재생성 강화** | 크로스레퍼런스 태그 존재 시 본문에 해당 키워드 없으면 재생성 | 기존 구조 활용 | 최대 재시도 횟수 제한 |
