# HIRA 약제 기준정보 통합 수집 계획

> 작성: 2026-04-27
> 대상 레포: scrape-hub (C:/Jimin/scrape-hub/)
> 연관: RegScan 엔리칭, dclaim-data-processor 이슈브리핑

## 1. 배경

RegScan 기사 파이프라인에서 글로벌 규제 뉴스(NICE, FDA, EMA 등)의 국내 관점 연결이 약함.
근본 원인: 국내 약제 급여 심의/평가/목록 데이터가 수집되지 않아 크로스레퍼런스 불가.

현재 수집 중인 HIRA 데이터:
- 보험인정기준 (5개 카테고리: 고시, 행정해석, 심사지침, 심의사례공개, 심사사례지침)
- 공지사항

수집하지 않는 데이터:
- **약제 기준정보 > 목록표** 전체 (5개 게시판)
- **약제 기준정보 > 암질환 사용약제 및 요법** 전체 (4개 게시판)

## 3. 기술 구조

### 공통 매커니즘

모든 게시판이 동일한 HIRA BBS 프레임워크 사용:
- URL: `https://www.hira.or.kr/bbsDummy.do?pgmid={pgmid}`
- 상세: `?pgmid={pgmid}&brdScnBltNo={scn}&brdBltNo={id}&pageIndex={page}`
- 테이블 기반 목록 (번호/제목/날짜/첨부)
- 페이징: 10/20/30건 선택 가능

기존 HIRA 보험인정기준 크롤러(`scrape-hub/project/hira/` 또는 `dclaim-data-collector/src/hira/`)와 **동일한 매커니즘**, pgmid만 다름.

### 수집기 설계

```
scrape-hub/project/hira_drug_info/
├── collector.py          # 통합 수집기 (pgmid 파라미터화)
├── config.py             # 9개 게시판 pgmid + 메타 정의
├── parser.py             # 목록/상세 파싱 (공통)
└── main.py               # 엔트리포인트
```

**핵심 설계 원칙:**
- 하나의 수집기 클래스로 9개 게시판 커버 (pgmid만 바꾸면 됨)
- **Forward only**: 전체 과거 데이터 안 긁음. 마지막 수집 시점 이후 신규 건만 수집
- 월 1회 스케줄 (cron 또는 수동)
- 기존 hira_master(235M)와 합쳐서 마스터오브마스터 구성 가능

## 4. 활용 경로

### A. RegScan 기사 엔리칭
- 글로벌 기사에서 약물 INN 감지 → 약제급여평가위 검색 → "2025년 제11차 급여 결정" 자동 연결
- 항암 기사 → 암질환 공고 검색 → "국내 항암 급여 공고 제2026-88호" 연결
- 약가/급여 기사 → 약제급여목록표에서 현재 등재 여부 확인

### B. 이슈브리핑 (dclaim-data-processor)
- 급여 공고/공고예고 → 이슈 해설 노트 자동 생성
- 신구조문대비표 추출 대상 확대

### C. 검색/QA
- "키트루다 담도암 급여 심의 결과" 같은 질문에 직접 답변 가능

## 5. 주의사항

- **심의 통과 ≠ 최종 급여**: 약제급여평가위 통과 후 약가 협상 → 최종 고시까지 갭 있음. 기사에서 "급여 결정"이 아니라 "급여 평가 통과"로 정확히 표현해야 함.
- **첨부파일**: 공고 상세에 HWP/PDF 첨부 많음. 파싱은 2차 과제 (Upstage API 활용 가능).
- **크롤링 부하**: 9개 게시판 × 월 1회면 부하 미미. 초기 1회 전체 수집 시에만 주의.
