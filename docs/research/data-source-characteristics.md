## 2. 소스별 상세 분석

### 2.1 PMDA (일본)

#### 승인 목록 (Excel)

| 항목 | 값 |
|------|-----|
| 수집 방식 | 연도별 Excel 다운로드 + 파싱 |
| 총 건수 | 275건 (2년, 2024-05 ~ 2026-03) |
| 고유 승인일 | **20일** (2년간) |
| 승인 간격 | 평균 **36일**, 중앙값 35일, 최소 8일, 최대 65일 |
| 1회당 건수 | 7~33건 (대량 배치) |
| 최신 데이터 | 2026-03-23 |

**갱신 패턴:** 월 1회 일괄 승인. 한 번에 20~30건을 몰아서 승인.
일본 후생노동성(MHLW) 약사분과회 심의 → PMDA 승인 → Excel 반영.

**승인일별 분포 (2025년도):**
```
2025-05-19:  10건    2025-08-08:   1건    2025-11-20:   7건
2025-06-24:  19건    2025-08-25:  12건    2025-12-22:  22건
                     2025-09-19:  33건
```

**파이프라인 권장:** `days_back=45` (월 1회 + 여유). 주간 실행 OK, 일일도 누락 없음.

#### RSS (심사/안전성)

| 항목 | 값 |
|------|-----|
| 심사 RSS (rss_011) | 17건/년, 최신 2026-04-14 |
| 안전성 RSS (rss_013) | 11건/년 |
| HTML 안전성 테이블 | 75건 (전체 이력) |

**갱신 패턴:** RSS는 주 1~2회 업데이트. Excel보다 빠른 선행 알림 역할.

**전략:** RSS(실시간 알림) + Excel(구조화 데이터) 조합.
RSS에서 "[SHINSA]" 태그로 신약 승인 뉴스를 먼저 감지, Excel에서 상세 데이터 확보.

### 2.2 NICE HTA (영국)

| 항목 | 값 |
|------|-----|
| 수집 방식 | storyblok CDN Excel 벌크 |
| 총 건수 | 179건 (최근 2년) |
| 전체 TA 이력 | 1,486건 (2000년~현재) |

**갱신 패턴:** NICE TA는 **개별 건 발행 시 즉시 반영**되지만, Excel 파일의 CDN 갱신 주기는 불명확.
TA 권고는 월 2~5건 발행 (연간 60~80건 수준).

**파이프라인 권장:** `years_back=2`로 전체 다운로드 후 기존 대비 diff. 일일 실행해도 트래픽 부담 적음 (단일 Excel).

### 2.3 MFDS 안전성 서한

| 항목 | 값 |
|------|-----|
| 수집 방식 | nedrug.mfds.go.kr 크롤링 (httpx+bs4, TLS 1.2) |
| 수집 건수 | 2건 (90일 기준, 테스트 시점) |
| 발행 빈도 | **비정기**, 월 0~3건 |

**갱신 패턴:** 안전성 이슈 발생 시에만 발행. 예측 불가.
2026-02-05, 2026-02-06 이후 4월까지 신규 없음 — 안전성 서한은 드물게 발행됨.

**주의:** nedrug.mfds.go.kr는 TLS 1.3 미지원 → `ssl.maximum_version=TLSv1_2` 필수.

**파이프라인 권장:** `days_back=30` 일일 실행. 데이터 없어도 정상 (0건 반환).

### 2.4 MOHW 건강보험정책 (건보심)

| 항목 | 값 |
|------|-----|
| 수집 방식 | MOHW 보도자료 게시판 키워드 필터 (httpx+bs4) |
| 수집 건수 | 2건 (365일 기준, 정밀 필터 후) |
| 관련 담당부서 | 보험급여과, 약무정책과 |

**갱신 패턴:** 건보심 안건은 **분기 1~2회** 정도 보도자료로 공개.
건강보험 관련 보도 자체는 월 5~10건이지만, 정밀 필터(담당부서+고신뢰 키워드) 적용 시 실제 급여 결정 관련만 추출.

**데이터 특성:**
- 건보심 의결 결과 → 보도자료로만 공개 (별도 게시판 없음)
- "보험급여과", "보험약제과" 담당 보도자료가 핵심 신호
- "급여" 단독 키워드는 복지급여/장애급여 등 노이즈 유발 → 제거함

**파이프라인 권장:** `days_back=30` 일일 실행. 건수가 적으므로 부하 없음.

### 2.5 국회 의안정보

| 항목 | 값 |
|------|-----|
| 수집 방식 | 열린국회정보 API (JSON) |
| 수집 건수 | 175건 (365일, 22대 국회) |
| 월평균 | **13.5건/월** |

**월별 분포:**
```
2025-04:  3    2025-08: 21    2025-12: 17    2026-04: 12
2025-05:  5    2025-09: 25    2026-01: 12
2025-06:  7    2025-10:  8    2026-02: 14
2025-07:  5    2025-11: 20    2026-03: 26
```

**갱신 패턴:** 국회 회기에 따라 변동 큼.
- 정기국회(9~12월): 월 17~25건 (집중기)
- 임시국회/휴회기: 월 3~8건
- 법안 발의는 수시 (의원 개인 발의)

**데이터 특성:**
- 16개 보건의료 키워드 검색 (약사법, 건강보험법, 의료법 등)
- 약사법 26건, 건강보험법 24건, 의료법 20건이 주축
- `proc_result` 필드로 처리 결과 추적 가능 (대부분 pending)
- User-Agent 헤더 필수 (없으면 400)

**파이프라인 권장:** `days_back=30` 일일 실행. 법안 발의 즉시 API에 반영됨.

### 2.6 FDA Orange Book / Purple Book

| 항목 | Orange Book | Purple Book |
|------|------------|-------------|
| 수집 방식 | CSV 다운로드 | CSV 다운로드 (세션 쿠키) |
| 전체 건수 | ~34,000 제품 | ~2,173 BLA |
| 갱신 주기 | **월간** (매월 1일 경) | **분기** |
| 핵심 데이터 | 특허 만료일, 독점권 | 독점권 만료, 바이오시밀러 |

**갱신 패턴:**
- Orange Book: FDA가 매월 초 CSV/ZIP 업데이트. 특허 만료일은 고정값이므로 한 번 수집하면 변동 적음.
- Purple Book: 분기별 갱신. Akamai bot 감지로 세션 쿠키 필요.

**파이프라인 권장:** 월 1회 전체 다운로드. 일일 실행 시에도 캐싱으로 불필요한 재다운로드 방지 권장.

### 2.7 EMA (유럽)

| 항목 | 값 |
|------|-----|
| 수집 방식 | JSON Report API |
| 전체 건수 | ~1,800+ 의약품 |
| 갱신 주기 | **주 1~2회** |

**갱신 패턴:** EMA는 매주 목요일에 CHMP 의견 발표, 이후 EC 결정으로 확정.
JSON report는 주 1~2회 갱신. `last_updated_date`, `european_commission_decision_date` 필드로 최근 변경분 필터 가능.

---

### 2.8 FDA (openFDA)

| 항목 | 값 |
|------|-----|
| 수집 방식 | REST API (drugsfda.json) |
| 갱신 주기 | **수시** (승인 즉시) |

**갱신 패턴:** FDA는 승인 즉시 openFDA에 반영. PDUFA date 기준으로 예측 가능.
AdCom, Safety 정보도 수시 업데이트.

---

### 2.9 DART 전자공시 / KIPRIS 특허 (미검증)

| 항목 | DART | KIPRIS |
|------|------|--------|
| 수집 방식 | REST API (JSON) | REST API (XML) |
| 갱신 주기 | **수시** (공시 즉시) | **수시** (공개/등록 즉시) |
| API 키 | 미등록 (구현 완료, 테스트 대기) | 미등록 (구현 완료, 테스트 대기) |

**예상 특성:**
- DART: 상장사 공시는 장 마감 후 집중 (17시 이후). 라이선스/임상 관련은 월 수 건.
- KIPRIS: 특허 공개는 출원 후 18개월, 등록은 수시. 의약품 IPC(A61K/A61P) 필터로 관련 특허만 추출.

## 4. 데이터 신선도 매트릭스

소스가 현실 세계의 이벤트를 반영하기까지 걸리는 시간:

```
이벤트 발생                    데이터 반영 시점
──────────                    ──────────────

FDA 승인                      → openFDA: 즉시 (~수시간)
                              → Orange Book: ~1개월 (월간 CSV)

EMA CHMP 의견                 → JSON Report: ~1주
EC 최종 결정                  → JSON Report: ~1주

MFDS 허가                     → 공공데이터 API: ~1주
MFDS 안전성 이슈              → nedrug 안전성 서한: ~수일
MFDS 회수 명령                → data.go.kr API: ~1~2일

PMDA 승인                     → RSS: ~수일 (속보)
                              → Excel: ~수주 (정확한 지연 불명)

NICE TA 발행                  → storyblok Excel: ~수일

건보심 의결                   → MOHW 보도자료: ~1~3일

법안 발의                     → 열린국회정보 API: 즉시 (~당일)

DART 공시                     → API: 즉시 (~당일)

특허 공개/등록                 → KIPRIS: 즉시 (~당일)
```

## 5. 주의사항

### 네트워크/인프라

| 이슈 | 대상 | 해결 |
|------|------|------|
| TLS 1.3 미지원 | nedrug.mfds.go.kr, pmda.go.jp | `ssl.maximum_version=TLSv1_2` |
| User-Agent 필수 | open.assembly.go.kr | 헤더에 `User-Agent` 추가 |
| Bot 감지 (Akamai) | purplebooksearch.fda.gov | 세션 쿠키 선획득 |
| API 키 만료 | data.go.kr | 주기적 갱신 필요 (현재 401) |

### 데이터 품질

| 이슈 | 대상 | 영향 |
|------|------|------|
| 날짜 형식 불일치 | PMDA Excel (YYYY.M.DD) | 파서에서 정규화 처리 |
| 한글/일본어 인코딩 | PMDA, MFDS | UTF-8 강제 |
| MOHW 키워드 노이즈 | 건강보험 보도자료 | 담당부서 필터로 완화 |
| NICE 날짜 없음 | Excel에 날짜 컬럼 불명확 | 전체 다운로드 + diff 전략 |
| PMDA Excel 갱신 시점 | 승인 후 수주 뒤 반영 | RSS로 선행 감지 보완 |
