## 3. 남은 문제

### 3.1 편집장 판단력

**핵심 문제: 편집장이 데이터 빈약 소스를 제외하지 않는다.**

| 소스 | 시그널 수 | 기사 가치 | 편집장 판단 |
|------|----------|----------|-----------|
| PMDA 승인 | 20건 | 높음 | 매번 선택 (정상) |
| 국회 법안 | 25건 | 높음 | 매번 선택 (정상) |
| KIPRIS 특허 | 20건 | 중간 | v4에서 선택, v6에서 누락 (불안정) |
| MOHW 건강보험 | **2건** | **낮음** | v6에서 2건을 2개 기사로 분리 (**과잉**) |

**원인:** 프롬프트에 "데이터 빈약 소스 제외"라고 했지만 LLM이 일관되게 지키지 않음. 매 실행마다 결과가 달라지는 비결정적 문제.

**대안:**
1. 코드에서 최소 시그널 수 미달 소스를 편집장에게 아예 안 넘기기
2. 편집장 출력을 검증하는 **가드레일 코드** 추가 (같은 소스 2건 → 자동 병합)

### 3.2 메타 언급

v6에서 "기사 초안은~", "초안에 따르면~" 같은 **자기 참조** 발생.
원인: 팩트체커→편집자로 넘어가면서 "초안"이라는 단어가 컨텍스트에 남아있음.

**대안:** 편집자 프롬프트에 "초안, draft, 원문 등 메타 단어 사용 금지" 추가.

### 3.3 기관명 일관성

"(이하 PMDA)" 지시를 했지만 LLM이 때로는 적용, 때로는 누락.
v5: 누락, v6: 일부 적용.

**대안:** 코드에서 후처리 — 기관명 패턴 매칭으로 자동 치환.

### 3.4 마무리 문장

"추가 확인이 필요하다", "~으로 정리됐다" 식 약한 마무리가 간헐적 발생.
팩트체커 체크리스트에 있지만 100% 제거 안 됨.

**대안:** 코드에서 금지 표현 regex 후처리.

## 4. 아키텍처 평가

### 4.1 프롬프트만으로 해결 가능한 것 vs 코드가 필요한 것

| 문제 | 프롬프트 | 코드 | 권장 |
|------|---------|------|------|
| 기사 톤/문체 | O | - | 프롬프트 |
| 구조 (리드→사례→맥락→전망) | O | - | 프롬프트 |
| 결측/지시문 제거 | △ (간헐적 누락) | O (regex) | **코드 후처리** |
| 기관명 표기 통일 | △ (불안정) | O (치환) | **코드 후처리** |
| 소스 다양성 | △ (불안정) | O (필터) | **코드 전처리** |
| 데이터 빈약 소스 제외 | X (안 지킴) | O (임계값) | **코드 전처리** |
| 메타 언급 제거 | △ | O (regex) | **코드 후처리** |

### 4.2 권장 아키텍처 변경

```
현재:
  시그널 전체 → 편집장(LLM) → 기자(LLM) → 팩트체커(LLM) → 편집자(LLM)

권장:
  시그널 → [코드: 최소 시그널 필터] → 편집장(LLM)
  → [코드: 소스 중복 검증] → 기자(LLM)
  → 팩트체커(LLM) → 편집자(LLM)
  → [코드: 금지표현 regex + 기관명 치환 + 메타언급 삭제]
```

코드 전처리/후처리로 **LLM의 비결정성을 보정**하는 구조.

## 5. 최선 기사 vs 최악 기사

### 5.1 최선 (v4 KIPRIS 기사)

```
# 면역항암 특허 21일간 4건 연속 공개

2026년 3월 27일부터 4월 16일까지 공개된 국내 특허 20건을 집계한 결과,
면역항암 관련 출원이 21일 동안 4건 연속 확인됐다. 이들 특허는 이중특이 항체,
면역표적 항체, 병용 조성물로 구성돼 같은 기간 면역항암 접근법이 한 축으로
수렴하기보다 분화돼 전개되고 있음을 보여준다.
```

**좋은 이유:** 테마 명확, 시간 순서 스토리, "분화돼 전개되고 있음을 보여준다" 의미 압축.

### 5.2 최악 (v6 MOHW #4 기사)

```
# 4/20 급여·약무 시그널 2건, 1건은 '포장'

이번 기사에서는 초안이 제시한 날짜·부서·공개 제목을 중심으로
정책 메시지를 재구성했으며, 세부 내용의 효과나 배경에 대한 해석은
포함하지 않았다.
```

**나쁜 이유:** 데이터 2건으로 억지 기사, 메타 언급, 의미 없음.

## 7. 다음 단계 권장

### 즉시 (코드 레벨)
1. **전처리:** 시그널 5건 미만 소스는 편집장에게 안 넘기기
2. **중간 검증:** 편집장 출력에서 같은 소스 2건 이상 → 자동 병합
3. **후처리:** 금지 표현 regex 삭제 + 기관명 자동 치환 + 메타 언급 제거

### 단기 (프롬프트 레벨)
4. 편집자에 "초안, draft" 등 메타 단어 금지 추가
5. 기자에 "~으로 정리됐다" 마무리 금지 강화
6. 레퍼런스 기사 Few-shot 예시 추가 (약업신문/뉴시스 본문 일부)

### 중기 (아키텍처)
7. KHIDI 제약산업정보포털 소스 추가 (30,439건 뉴스 아카이브)
8. 주간 종합 리포트 구현
9. 기사 발행 채널 연동 (이메일, 대시보드, API)

## 8. 결론

6번의 반복으로 **"내부 메모" → "산업 기사에 근접한 수준"**까지 도달.
프롬프트만으로는 LLM 비결정성 한계가 있어, **코드 전처리/후처리 레이어**가 필수.
데이터 소스 품질(시그널 수)이 기사 품질을 결정하는 핵심 변수.
PMDA(20건), 국회(25건), KIPRIS(20건)은 기사 가치 충분.
MOHW(2건)은 기사 단독 발행에 부적합 — 다른 소스와 병합 필요.
