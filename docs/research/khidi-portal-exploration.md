> 조사일: 2026-05-11
> 목적: KHIDI 제약산업정보포털(`/epharmakorea`) 미수집 게시판 식별 및 XML API 검증

---

## 1. 포털 구조

KHIDI(한국보건산업진흥원)의 제약 관련 데이터는 3개 채널로 분산:

| 채널 | URL | 현재 수집기 |
|------|-----|------------|
| KHIDI 본관 (산업동향) | `/board?menuId=MENU01783` | `khidi.py` (브리프/보고서) |
| 제약산업정보포털 (뉴스) | `/board?menuId=MENU01816` | `khidi_news.py` (국내/해외뉴스) |
| 제약글로벌정보센터 | `/kps/openAPI/requestxml` | `khidi_global.py` (약가·보험, 법령·고시) |

**사업포털** (`/menu?menuId=MENU03862`)에서 15개 하위 포털 링크 제공.
그 중 **제약산업정보포털** (`/epharmakorea`)이 제약글로벌정보센터를 포함하는 본체.

## 3. XML API 검증 결과

### API 엔드포인트

```
GET https://www.khidi.or.kr/kps/openAPI/requestxml
  ?menuId=MENU#####
  &rowCnt=50        # 페이지당 건수
  &pageNum=1        # 페이지 번호 (선택)
```

### 응답 구조 (모든 게시판 동일)

```xml
<openAPI>
  <totalCnt>530</totalCnt>
  <row>
    <title>제목</title>
    <date>2026-05-06</date>
    <linkid>12345678</linkid>
    <titleid>67890</titleid>
    <boardid>BOARD01234</boardid>
    <menuid>MENU02599</menuid>
    <username>관리자</username>
    <url>https://...</url>
    <content>본문 HTML</content>
  </row>
  ...
</openAPI>
```

## 5. 활용 방향

### 기사 파이프라인 시그널 강화

| 게시판 | 활용 | 시그널 유형 |
|--------|------|------------|
| 시장 현황 | 시장규모/트렌드 맥락 | 산업 배경 |
| 허가 및 특허 | 신규 승인 속보 | 규제 이벤트 |
| R&D 정보 | 딜/파이프라인 뉴스 | 기업 동향 |
| 임상 및 비임상 | 임상시험 주간 현황 | 개발 파이프라인 |

### MedicinePage 약물 카드 보강

- 허가 및 특허 → 약물별 글로벌 허가 타임라인
- R&D 정보 → 라이선스 딜, 파이프라인 업데이트
- 임상 및 비임상 → 약물별 임상시험 진행 현황

### 레퍼런스 자료

- 진흥원 보고서 → 산업 분석 리포트
- 유관기관 보고서 → 시장전망, R&D 투자 현황
- 통계간행물 → 연간 통계집, 시장규모 시계열
- 전문가 Insight → 국가별 인허가 실무 가이드

## 6. 구현 방침

기존 `khidi_global.py`의 `GLOBAL_BOARDS` dict에 10개 게시판 추가.
XML API 구조가 완전 동일하므로 코드 변경 최소화.
`source_type`은 `KHIDI_GLOBAL_INFO` 유지, `board` 필드(label)로 카테고리 구분.
