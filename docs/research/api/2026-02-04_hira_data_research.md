## 데이터 연계 전략

### MFDS ↔ HIRA 매칭

```
MFDS 허가정보           HIRA 적용약가
├── ITEM_SEQ ──────────→ 품목코드 (직접 매칭 시도)
├── ITEM_INGR_NAME ────→ 주성분코드 (성분명 매칭)
└── ITEM_NAME ─────────→ 품목명 (제품명 매칭)
```

### GlobalRegulatoryStatus 통합

```python
@dataclass
class GlobalRegulatoryStatus:
    inn: str                    # 국제 성분명
    fda: FDAStatus | None       # FDA 승인 정보
    ema: EMAStatus | None       # EMA 승인 정보
    mfds: MFDSStatus | None     # MFDS 허가 정보
    hira: HIRAStatus | None     # NEW: HIRA 급여 정보

@dataclass
class HIRAStatus:
    item_code: str              # 품목코드
    item_name: str              # 품목명
    reimbursement: str          # 급여/삭제/급여정지/선별급여
    price_ceiling: int          # 상한가
    ingredient_code: str        # 주성분코드
    manufacturer: str           # 업체명
    apply_date: date            # 적용일자
```

### 탐지 시나리오

| 시나리오 | 조건 | 의미 |
|---------|------|------|
| 국내 도입 임박 + 미급여 | FDA/EMA 승인 + MFDS 허가 + HIRA 미등재 | 급여 등재 예상 |
| 글로벌 승인 + 국내 미허가 | FDA/EMA 승인 + MFDS 미허가 | 허가 신청 예상 |
| 선별급여 대상 | HIRA 선별급여 | 본인부담률 높음 |
| 급여 삭제 | HIRA 삭제 | 더 이상 급여 안됨 |
