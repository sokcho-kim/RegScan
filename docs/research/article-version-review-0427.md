## 3. 편차 원인 분석

### 문제 1: LLM 출력 분산 — 같은 입력에서 품질 편차

**증거:**
- V9 마약류 기사: 구매→보관→사용→기록→폐기 전 과정 구체적
- V11 마약류 기사: "강화 추진"만 반복, 내용 얇음
- 동일 프롬프트, 동일 데이터, 다른 결과

**원인:** LLM(GPT-5.2)의 temperature=0.3에서도 출력 분산이 존재. 기자 프롬프트가 "이렇게 써라"는 구조를 주지만, 데이터 해석 깊이는 매번 다름.

**대책 후보:**
- temperature 0.1~0.2로 낮추기 (분산 축소, but 다양성도 줄어듦)
- 품질 검증에서 "구체성 체크" 추가 (키워드 밀도 등)
- 최선 선택: 동일 소재로 2~3개 초안을 뽑고 가장 좋은 것을 선택 (비용 증가)

### 문제 2: 크로스 소재 연결이 비결정적

**증거:**
- V9 CSU 기사: NICE TA969 비제출과 자연스럽게 연결
- V11/V12 CSU 기사: NICE 연결 없음, 단독 기사로 처리

**원인:** 편집장이 "이 소재끼리 엮어라"를 지시할 수도 있고 안 할 수도 있음. 프롬프트에 크로스레퍼런스를 강제하는 규칙이 없음.

**대책 후보:**
- 편집장 프롬프트에 "같은 약물/치료영역이 2개 이상 소스에 나오면 반드시 연결" 강제
- 시그널 전처리에서 약물명/치료영역 기준 클러스터링 후 편집장에게 전달
- 기자 프롬프트에 "다른 기사 스토리와의 연결점이 있으면 반드시 언급" 추가

### 문제 3: 엔리칭 데이터가 기사에 반영 안 됨

**증거:**
- V10/V11/V12에서 국내 크로스레퍼런스 엔리칭이 추가됨
- 그런데 기사 본문에 국내 허가/급여/임상 정보가 하나도 안 나옴

**원인 추정:**
1. 엔리칭이 NICE_TA 시그널에 붙긴 했는데, 기자가 `fda_context` 필드를 무시
2. 또는 NICE 시그널에 약물 INN이 직접 포함되지 않아서 `_extract_inns_from_text`가 매칭 실패
3. 또는 format_for_prompt에서 fda_context가 잘리거나 누락

**확인 필요:** 엔리칭 로그에서 NICE_TA에 크로스레퍼런스가 실제로 붙었는지 확인

### 문제 4: 팩트체커 과잉 폐기 (V10)

**증거:** V10에서 3건 중 2건 폐기. V9에서는 같은 소재가 통과.

**원인:** kill_keywords가 너무 넓음. "확인 불가"가 "이 사실이 거짓"이 아니라 "체크리스트 항목을 확인할 수 없다"는 뜻으로도 트리거.

**대책:** V12에서 보강 루프로 해결 완료 — 즉사 제거, 3회 재작성 기회, 등급 다운그레이드.

## 4. 각 버전의 "가장 좋았던 것"

| 버전 | Best | 이유 |
|------|------|------|
| V5 | NICE 정보 풍부 | 15건 중 5건 + 권고 9건 병기, 전체 그림 |
| V8 | 문체 개선 | 간결하고 뉴스체에 가까워짐 |
| V9 | **CSU 기사 전체** | NICE 크로스 + 국내 쟁점 3개 — 전 버전 통틀어 최고 |
| V9 | 마약류 기사 구체성 | 전 과정 서술이 B등급 해설의 정석 |
| V10 | 보강 루프 도입 계기 | 이 실패가 즉사 제거 + 보강 루프 설계로 이어짐 |
| V11 | **NICE 구조 분석** | 2.8% 비율, 항암/비항암 분류 — 분석 기사의 정석 |
| V12 | 기사 생존율 | 보강 루프 작동으로 3건 전부 생존 |

## 5. "이상적 기사"의 조합

현재까지의 최고 요소를 조합하면:

### NICE 기사 이상형
- V11의 구조 (비율 + 항암/비항암 분류)
- V5의 정보량 (전체 스코프 + 권고 건수 병기)
- V9에서 보여준 크로스레퍼런스 능력 (다른 기사와 연결)
- **+ 국내 관점** (키트루다 국내 급여 현황, 비제출이 국내 등재 전략에 시사하는 점)

### CSU 기사 이상형
- V9의 NICE TA969 연결
- V9의 국내 쟁점 3개 (비교약, 단계치료, 비용-효과)
- V11의 메시지 분석 (노바티스 포지셔닝 관점)
- **+ 오말리주맙 국내 급여 현황** (엔리칭 데이터 활용)

### 국내 소재 이상형
- V9의 구체성 (전 과정 서술)
- V12의 팩트 밀도 (수치, 날짜, 후속 조치)
- 카드형이 아닌 해설형 유지 (화면 풍성함)
